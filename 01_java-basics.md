# 第1章：Javaの基本ルールと考え方

プログラミングの基本（変数にデータを入れる、条件によって処理を変えるなど）は、どの言語でも共通しています。その経験はJavaでも必ず活きます。
ここからは、「Javaという言語がどのようなルールで動いているのか」という特有の考え方を学びます。

## この章でできるようになること

- Javaで「なぜコンパイルが必要か」を、品質と安全性の観点で説明できる
- 型（Type）の役割を理解し、基本的な変数宣言とメソッド定義を書ける
- 文法エラーと設計上の意図を切り分けて考えられる

## 1. コードが動くまでの手順（コンパイルという安全装置）

授業などで触れてきた多くのプログラミング言語では、コードを書いたらすぐに実行できるものが多いです（インタープリタ方式）。
しかしJavaは、実行する前にプログラム全体を一度機械が読みやすい形式に翻訳する **「コンパイル」** という手順が必ず必要です。

- **なぜわざわざ手間をかけるのか？**  
  コンパイルを行うタイミングで、Javaはコードの文法ミスや矛盾をチェックしてくれます。つまり、**「プログラムを動かしてエラーで止まってしまう前に、間違いを教えてくれる安全装置」** が備わっているということです。

## 2. Javaの最大の特徴「型（Type）」の厳格さ

Javaを学ぶ上で最も重要になるのが、変数の **「型（Type）」** という概念です。

プログラミングにおける変数を「箱」だと想像してください。
動的型付けの言語（授業などで触れた言語の多くがこれに該当します）では、１つの変数に数値でも文字列でも何でも自由に入れることができます。しかしJavaでは、**変数の箱を用意する際に「この箱には〇〇専用のデータしか入れません」と厳格に宣言する** 必要があります。（これを静的型付けと呼びます）

```java
// String（文字列）専用の箱を用意し、文字を入れる
String data = "こんにちは";

// int（整数）専用の箱を用意し、数値を入れる
int number = 100;

// data = 100; // ← [エラー！] 文字専用の箱に数値は入れられない！
```

- **なぜこんなに厳しいルールがあるのか？**  
  システム開発は何人ものチームで行います。もし「数値が入るはずの箱に、他の誰かが間違って文字を入れてしまった」場合、システムは計算ができずにクラッシュしてしまいます。
  Javaは箱の用途を厳格に決めることで、間違ったデータが入ろうとした瞬間、動かす前の「コンパイル」の段階でエラーを出して開発者に警告してくれます。これがシステム開発において絶大な安心感を生みます。

## 3. Javaの記述ルール（見た目の約束事）

- **文の終わり**：日本語の句点（。）のように、処理の終わりには必ず **セミコロン（ `;` ）** を書きます。
- **まとまり（ブロック）**：条件分岐などで「ここからここまで」という処理のまとまりは、**波括弧（ `{}` ）** で囲みます。
- **スタート地点**：Javaのプログラムは、必ず `public static void main(String[] args)` という決まったブロックから処理がスタートします。

## 4. パッケージ（クラスを整理するフォルダ）

Javaの開発では、ファイル（クラス）の数が数十、数百と増えていきます。これらが同じ場所に混在すると、管理が大変になったりファイル名が衝突したりします。
それを防ぐ仕組みが **パッケージ（package）** です。

PCの「フォルダ」と同じように、関連するクラスをまとめる役割を持ちます。Javaのソースコードの **一番最初の行** には、必ず「このクラスはどのパッケージに属しているか」を宣言します。

```java
package com.example.demo; // このファイルが置かれている場所（フォルダ階層）を示す

public class HelloWorld {
    // ...
}
```

後の章で学ぶSpring Bootの実務では、このパッケージをどう分けるか（Controller用のフォルダ、Service用のフォルダなど）がシステム構造そのものになります。今は「1行目の package はファイルの置き場所を示している」と覚えておきましょう。

## 5. メソッド（クラスが持つ機能・振る舞い）

プログラミングをしていると、ひとまとまりの処理に名前をつけて呼び出したくなります。他の言語ではよく「関数（Function）」と呼ばれます。
一方、オブジェクト指向言語であるJavaでは、プログラムを構成する「部品」はあくまで **クラス** であり、そのクラスが持っている **機能や個別の振る舞い** のことを **「メソッド（Method）」** と呼びます。メソッド＝独立した部品というよりは、「クラスという部品がもつ機能」と捉えると、後の概念と混同しません。

メソッドを定義するには、以下の要素を宣言します。

- **戻り値の型**：メソッドが処理後に返すデータの型（`int`、`String` など）。何も返さない場合は特別なキーワード **`void`** を書きます。
- **メソッド名**：呼び出す際に使う名前。
- **引数（パラメータ）**：メソッドへの入力データ。不要な場合は `()` の中を空にします。

```java
// [修飾子] [戻り値の型] [メソッド名]([引数の型 引数名]) { 処理 }

// 例1：int型を2つ受け取り、合計値（int型）を返すメソッド
public static int add(int a, int b) {
    return a + b; // return で呼び出し元に値を返す
}

// 例2：引数なし・戻り値なし（void）のメソッド
public static void printMessage() {
    System.out.println("処理が完了しました！");
    // void の場合、return は不要
}
```

> **`static` について**：`main` メソッドに付いている `static` と同じキーワードです。`main` から呼び出すメソッドには同じく `static` を付ける必要があります。`static` の詳しい意味は[第2章](02_oop-fundamentals.md)の「4. `static` とインスタンスメソッドの違い」で改めて理解できます。

定義したメソッドを「呼び出す」には、メソッド名と引数を書くだけです。

```java
int result = add(3, 5);  // add メソッドを呼び出し、戻り値を result に受け取る
// result の中身は 8 になる
printMessage();           // printMessage メソッドを呼び出す
```

メソッドを使うことで、「同じ処理を何度もコピーして書く」という無駄がなくなり、修正も1箇所だけで済むようになります。

### 【サンプルコード】Javaの基本の形

以下のコードは、完全に動作するJavaのプログラムです。「型」の宣言、セミコロン、波括弧の使い方、そしてメソッドの定義と呼び出し方を確認してください。

**ファイル名: `HelloWorld.java`**

```java
public class HelloWorld {
    // ここがプログラムのスタート地点です
    public static void main(String[] args) {

        // 1. 変数の宣言（必ず型を書く）
        String greeting = "Hello, Java!"; // String は文字列型
        int year = 2026;                  // int は整数型

        // 2. コンソール（画面）への出力
        System.out.println(greeting);
        System.out.println("今年は " + year + " 年です。");

        // 3. 条件分岐（条件式は()で囲み、処理のブロックは{}で囲む）
        if (year > 2020) {
            System.out.println("新しい時代ですね！");
        }

        // 4. メソッドの呼び出し（このクラスで定義したメソッドを使う）
        int result = add(10, 20);
        System.out.println("10 + 20 = " + result);
        printSeparator();
    }

    // 【メソッドの定義①】int型を2つ受け取り、合計値（int型）を返す
    public static int add(int a, int b) {
        return a + b;
    }

    // 【メソッドの定義②】引数なし・戻り値なし（void）のメソッド
    public static void printSeparator() {
        System.out.println("--- 処理完了 ---");
    }
}
```
