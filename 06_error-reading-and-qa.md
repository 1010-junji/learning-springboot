# 第6章：現場での「質問の仕方」と「エラーの読み方」

プログラムを書いていれば、必ずエラーに遭遇します。ベテランのエンジニアでも1日に何十回もエラーを出します。
エラーは「敵」ではなく、システムがあなたに **「ここがおかしいから直して！」と教えてくれている「ヒント」** です。

## この章でできるようになること

- スタックトレースから「根本原因」と「調査開始位置」を特定できる
- 相談時に必要な情報（ゴール、事実、試したこと）を整理して伝えられる
- 詰まったときに、自己解決と質問の切り替え判断ができる

## 1. 魔法の暗号「スタックトレース」の読み解き方

Spring Bootでエラーが起きると、コンソール（実行画面）に何十行にもわたる赤い英語の文字の羅列が出力されます。これを **「スタックトレース（エラーの足跡）」** と呼びます。

初心者はこの文字量に圧倒されてフリーズしてしまいますが、**実は見るべきポイントは「3箇所」しかありません。**

**【見るべきポイント】**

1.  **一番上の行（エラーの種類）**：何が起きたのか？（例：データが空っぽだぞ！など）
2.  **`Caused by:`（根本原因）**：ログの下の方にあるこの文字を探します。「本当の原因はこれだよ」と教えてくれています。
3.  **自分が書いたパッケージ名（`com.example...` など）**：ログの中に、Springの内部コード（`org.springframework...`）がたくさん混ざっていますが、それは無視してOKです。**「自分が作ったファイルの何行目か」**を探します。

### よく見る代表的なエラー（これが出たらアタリ！）

- **`NullPointerException`（ヌルポ）**
  「箱（変数）の中身が空っぽ（Null）なのに、中身を使おうとしたぞ！」というJavaで最も有名なエラーです。
- **`BeanCreationException` / `NoSuchBeanDefinitionException`**
  Spring Boot特有のエラーです。「DI（注入）しようとしたけど、そんなクラス見つからないよ！」という意味です。（[**第4章**](04_di-and-annotations.md)で学んだ `@Service` などの**アノテーションの付け忘れ**が原因の9割です）。

## 2. 悪い質問と、良い質問の違い

自力でエラーログを読んでも解決できない時は、チームの先輩に質問します。
しかし、ここで「質問の仕方」を間違えると、解決に時間がかかるだけでなく、お互いにストレスが溜まってしまいます。

**【悪い質問（「わからないことがわからない」状態）】**

> 「先輩、ユーザー登録画面を作っていたんですが、なんか動きません。エラーが出てるんですけど、どうすればいいですか？」

- **なぜダメなのか？**：先輩はあなたのパソコンの前に座って、イチから状況をエスパー（推測）して調べ直さなければなりません。

**【良い質問（事実と自分の考えを整理している状態）】**

> 「先輩、ユーザー登録機能について質問です。
> **【実現したいこと】** 登録ボタンを押したら、DBにデータが保存されて完了画面が出るようにしたいです。
> **【現状起きている問題】** ボタンを押すと画面が真っ白になり、コンソールに `NullPointerException` が出ています。
> **【自分で調べたこと・試したこと】** エラーログを見ると `UserService.java` の25行目で落ちていました。ControllerからServiceにデータが渡っていない気がするのですが、解決方法がわかりません。5分ほどお時間いただけますでしょうか？」

- **なぜ良いのか？**：先輩は「あ、Serviceの25行目を見ればいいんだな。Controllerからのデータの渡し方が怪しいな」と、**一瞬でアタリをつけて回答（または一緒に調査）することができます。**

## 3. そのまま使える！質問テンプレート

現場で質問・チャットをする際は、必ず以下のテンプレートを埋めてから送信する癖をつけてください。これを埋める過程で「あ、ここを調べ忘れてた」と気づき、自己解決できることも多々あります。

```text
【件名】〇〇機能のエラーについての相談（〇〇分ほどお時間よろしいでしょうか）

【1. 実現したいこと（ゴール）】
・（例：一覧画面に、データベースから取得したユーザー名を10件表示したい）

【2. 現状起きている問題（事実）】
・（例：画面を開くと「500 Internal Server Error」になり、
　コンソールに NoSuchBeanDefinitionException が出ている）

【3. エラーの発生箇所（ログから特定した場所）】
・（例：UserController.java の 15行目付近）
・※必要であればエラーログのテキストを添付します。

【4. 自分で試したこと・推測】
・（例：ServiceクラスがDIできていないと思い、@Serviceアノテーションが
　ついているか確認しましたが、ついていました。他に原因が思い当たらず詰まっています）
```

## 確証をとるためのテスト（第6章）

現場に出る前の最終テストとして、以下の実践的なシチュエーションにどう対応するかを確認してください。

**【問題1：スタックトレースの解読】**  
プログラムを実行したところ、コンソールに以下のようなエラーログが大量に出力されました。

```text
org.springframework.beans.factory.UnsatisfiedDependencyException: Error creating bean with name 'userController' defined in file ...
    at org.springframework.beans.factory.support... (省略)
    ...
Caused by: org.springframework.beans.factory.NoSuchBeanDefinitionException: No qualifying bean of type 'com.example.demo.service.UserService' available: expected at least 1 bean which qualifies as autowire candidate.
    at org.springframework.beans.factory.support... (省略)
    at com.example.demo.controller.UserController.<init>(UserController.java:12)
    ...
```

このログから読み取れる **「エラーの根本原因」** と、**「エラーが発生しているあなたのコードのファイル名と行数」** を答えてください。また、このエラーを直すために、あなたは **どのファイルに何を書き忘れている可能性が高い** と推測しますか？（[**第4章**](04_di-and-annotations.md)の知識を使います）。

**【問題2：質問の組み立て】**  
あなたは先輩に「商品の一覧を表示する機能を作って」と指示されました。
しかし、`ItemRepository` を使ってデータベースからデータを取り出そうとすると、`NullPointerException`（データが空っぽのエラー）が出てしまい、20分調べて直りませんでした。
上記の「質問テンプレート」を使って、先輩にチャットで相談する文章を作成してください。（推測の部分は想像で構いません）。
